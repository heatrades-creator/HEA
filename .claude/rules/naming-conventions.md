# HEA Naming Conventions — Ecosystem Standard

## The Core Pattern

```
{JOB-ID}-{descriptor}-{Client-Name}-{YYYY-MM-DD}
```

All auto-generated content — folders, PDFs, photos, receipts, documents — follows this pattern.

- **Separator:** hyphens everywhere, no underscores, no spaces
- **Date:** `YYYY-MM-DD` always (ISO, sorts correctly)
- **Client name:** `Firstname-Lastname` (title-case each word, hyphens between words)
- **Descriptor:** lowercase, hyphenated words
- **Extensions:** lowercase, preserved from source

---

## Job ID Format

```
TS{NNNNN}   ← 5 digits, zero-padded, sequential
```

| Example | Meaning |
|---|---|
| `TS00001` | First job |
| `TS00012` | Twelfth job |
| `TS00100` | Hundredth job |

**Reset point:** Ecosystem starts at `TS00001` (live launch).  
**GAS generates job IDs** — the `getNextJobNumber()` function in `HEAJobsAPI.gs` is the single source of truth.  
**No other ID format** (JOB-YYYY-NNNN, HEA-NNNN, etc.) should appear in new content.

---

## Drive Folder — Client Job Folder

```
TS00001-John-Smith-2026-05-03/
```

Pattern: `{JOB-ID}-{Client-Name}-{intake-date}`  
Parent: `CLIENTS_FOLDER_ID`  
Date: date the job was created (intake date)

---

## Drive Folder — Standard Subfolders

Created inside every client job folder. Fixed names, do not vary:

```
00-nmi-data/       ← PowerCor NMI files
01-quotes/         ← Quote PDFs
02-proposals/      ← Proposal documents
03-signed/         ← Signed contracts
04-installed/      ← Post-install photos
05-photos/         ← All intake + installer photos
06-jobfiles/       ← Intake docs (consent PDF, job card, electricity bill)
07-receipts/       ← Job site receipts uploaded by installers
```

## Photos — Installer App Uploads (Post-Install)

All photos stored in `05-photos/` subfolder.

Pattern: `{JOB-ID}-photo-{category}-{Installer-Name}-{YYYY-MM-DD}.{ext}`

| Category | Slug |
|---|---|
| Switchboard | `switchboard` |
| Battery location | `battery-location` |
| Panel array | `panel-array` |
| Inverter | `inverter` |
| Meter box | `meter-box` |
| General site | `site` |

Example: `TS00001-photo-panel-array-Josh-Brown-2026-05-03.jpg`

---

## Receipts — Job Site (Installer App)

Stored in `07-receipts/` subfolder.

```
{JOB-ID}-receipt-{Installer-Name}-{YYYY-MM-DD}.{ext}
```

Example: `TS00001-receipt-Josh-Brown-2026-05-03.pdf`

If multiple receipts on the same day, append a counter: `-2`, `-3`, etc.

---

## Receipts — General Staff (No Job)

Stored in `--- Staff Receipts --- / {YYYY-MM} /`

```
receipt-{Staff-Name}-{YYYY-MM-DD}.{ext}
```

Example: `receipt-Josh-Brown-2026-05-03.pdf`

---

## Payment Request Records

Stored in the client job folder root.

```
{JOB-ID}-payment-{milestone}-{Client-Name}-{YYYY-MM-DD}.txt
```

| Milestone slug | Meaning |
|---|---|
| `deposit` | Initial deposit |
| `progress` | Progress payment |
| `final` | Final payment |

Example: `TS00001-payment-deposit-John-Smith-2026-05-03.txt`

---

## Quote Numbers (Solar Analyser — Pre-Job)

Quote numbers are issued before a job number exists. They remain as:

```
HEA-{YYYYMMDD}-{RRR}
```

Example: `HEA-20260503-427`  
Once a lead converts to a job, the job number (`TS00001`) becomes the primary reference.

---

## Archive / Organisational Folders (Unchanged)

These are Drive organisational folders, not job content. No change:

| Folder | Pattern |
|---|---|
| Monthly archive | `{YYYY-MM} - {MMMM YYYY}` e.g. `2026-05 - May 2026` |
| Monthly staff receipts | `{YYYY-MM}` e.g. `2026-05` |

---

## Sanitisation Rules

When building names from user input:

1. Trim leading/trailing whitespace
2. Remove characters: `/ \ ' " < > | * ? : @`
3. Collapse all whitespace runs to a single `-`
4. Remove consecutive hyphens (`--` → `-`)
5. Title-case each word in client/installer names
6. Max 50 characters per segment (job ID and date don't count toward this)

**GAS function:** `safeString_()` in `HEAJobsAPI.gs` handles rules 2–4. Wrap client name words in title-case before passing.
